## 1. 가장 중요한 발견 — 교육 방향 전환

인터뷰에서 드러난 실측 사실:

- **전원이 이미 자기 업무를 어느 정도 자동화하고 있다.** 경영진(위)에서 "네 업무를 최대한 자동화해봐라"라고 드라이브를 걸어 결과물을 다들 만들어 둔 상태.
- **그러나 "여정(과정)"을 모른다.** 문제 → 결과물 사이가 비어 있다. A부터 Z까지 이해하고 만든 게 아니라, 클로드에게 시켜서 결과만 뽑아냈다. "순간 모면식"으로 처리해 과정을 설명하지 못한다.
- AX 사다리로 보면 **결과물은 3~4단**을 내지만, **이해/여정은 0~1단**인 불일치 상태.

→ **결론(교육 재설계):**
- 당초 계획한 "예제 → 실전 실습 → 과제" 방식을 **버린다.** 이미 각자 실전 과제를 갖고 있으므로, 또 실습을 시키면 여정을 모른 채 결과만 또 뽑고 끝난다.
- 대신 **각자가 이미 만든 과제를 "이해하고 디벨롭"하는 방식**으로 전환:
  - AI/컴퓨터가 동작하는 원리
  - 클로드 초기 세팅(어떤 문서를 넣으면 왜 더 좋아지는지)
  - 프롬프팅, 페르소나, 스킬화, 봇 연결
  - "내가 만든 게 왜 저렇게 됐고, 어느 방향으로 디벨롭하면 되는지"에 설 수 있는 기초
- 원리는 도구가 바뀌어도 동일(ChatGPT·클로드 원리 같음) → 한 번 익히면 도구 전환 가능하다는 관점을 심는다.

## 3. 개인별 상세

### 서지애 과장 (경영기획팀 · 기초 트랙이나 실질 최상위)
- **현황:** 원래 책 위주 → 클로드 코워크 → 코드로 이관 중. AX 4단계(채팅-연결-MCP스킬-자동화) 개념을 이미 이해.
- **완료한 자동화:**
  1. **통장 자료 수집** — 매일 자금보고용. 크롬 자동로그인 → 통장자료 메뉴 → 엑셀 다운 → 실장님 메일 발송. 파이썬 + 작업 스케줄러. 최종확인자가 실장이라 파이썬 코드 전체를 실장 PC로 이관해 발행 → **해당 업무 소거 완료.**
  2. **건물 관리비 청구서** — 한전/수도 고지서 사용량 입력 → 단가 계산 → 임대료+관리비+전기세 청구서 생성 → PDF 변환. 자동화 완료.
- **진행 중:** 경영 대시보드 제작(실장 보고 前). Netlify(매트리파이) 배포했으나 유료 → **Vercel(버스) 전환 희망**(Git 연동 배포).
- **최종 목표(꿈):** 팀 데이터가 더존·엑셀·구글드라이브에 분산 → **3사 법인(신지·2C지·심플레이) 손익 동시 조회 + 상계 거래 제외** 보고. 매월 월결산 로우데이터를 규칙 기반 자동 변환. → **시스템화가 정답**(로컬 파이썬보다).
- **막힌 것 / 한계:**
  - 금융자산 현황 업데이트 — 금융사 보안 강화로 클로드 로그인 불가. 공동인증서/아이디·비번 방식은 위험(출금 리스크) → **한계로 결론.**
  - 더존 MCP 미제공 → 브라우저 RPA 클릭 다운로드가 유일. 원드라이브 경로 불일치 → 구글 스프레드시트로 우회.
- **팀원 현황:** 매출입금 + 국세청 세금계산서 매칭(외상매출금 잔액 입금 시점) — 구글 앱스스크립트로 진행 중.
- **질문:** 토큰 절약법 (→ 모델 선택[플랜 Opus/실행 Sonnet], 컨텍스트 최적화, 하네스 엔지니어링, CLAUDE.md 슬림화 안내).
- **선정 과제:** RPA로 더존 데이터 다운 → 스킬 호출로 정제 → 대시보드 반영/배포. 월 1회 비정기라 **대시보드에 "다운로드+반영" 버튼**으로 구현. Git↔Vercel 연결 배포.
- **교육 피드백:** 봇교육 IT 기초 부분이 거의 없이 답변 몇 개면 다음 단계로 넘어갔다(체감 부족).

### 신은선 과장 (홈쇼핑사업팀 · 심화)
- **현황:** 클로드로 업무, 일상질문은 Gemini. 봇교육 "진짜 개발자라 어렵다" — 서버 개념·코드 관리부터 어려움(재복습 필요).
- **완료한 자동화:**
  1. **일계표 작업** — 클로드가 엑셀 매크로 코드 작성 → 엑셀에 붙여 클릭만으로 자동.
  2. **매출 정리** — 세일코리아 주문매출 ↔ 자사 편성 매출 엑셀. 매핑 테이블(좌: 세일코리아 상품명, 우: 편성명)로 매칭 → 시간대별로 쪼개 상품 합계. "런(run)" 파일 더블클릭 시 자동 반영. 매일 아침. (세일즈오토 = 코드 생성용)
- **원하는 과제 → 선정:** **자사 상품 입고+입금 매칭 자동화.** 창고 입고 송장 고객명 ↔ 우리은행 입금자명 매칭(불일치 시 사람 보정) → 세일코리아 반품서 요청. 매일. 결과는 **네이트온으로 텍스트 복붙**(MCP 미제공 → 텍스트만 추출·누적). 이지어드민 엑셀 등록 가능.
- **기대:** 과제가 너무 쉽다 → 좀 더 하고 싶다. "어느 정도 자동화해보는 것."
- **비고:** 오늘 447건 주문이 세일코리아와 맞는지 검수 요청받음.

### 박성희 실장 (경영기획 · 심화) — 결과물 2~3단 / 이해 하 (녹취 없음, 요약 기반 확정 07-09)
- 개발 가능 여부에 관심. 법인 3개 · 부서 2개. 법인 로그인 재인증 이슈.

- 슬랙 사용. ~~교육봇 직접 제작~~ → **정정: 교육봇은 위시켓(진행자) 제작.** 실장은 사용자.
- 관심: 동작 원리 + 유용한 프롬프팅.
- **확정 평가:** 자동화 산출물을 실무에 운용 중이나 직접 제작·과정 이해는 제한적. **다음 스텝 = OMS/시스템 개발 참여.**

### 서희정 사원 (홈쇼핑사업팀 · 심화)
- **업무:** 상품 기획, 정산, 인포머셜 영상 검수.
- **도구:** 클로드(대부분)·Gemini·ChatGPT·젠스파크(이미지). 프로플랜(월 3만원) — 한도 금방 참. 학습은 책 위주.
- **완료한 자동화 (포트폴리오 3종):**

  2. **심의필증 검수** — 자막 대본 업로드 → 위반 표현(예: "최고") 탐지 → 업체 수정요청 메일 작성. 클로드로 사이트 제작, URL 팀 공유.

### 박정선 팀장 (홈쇼핑사업팀 · 심화, 팀 리더)

- **걱정:** 파일·이메일 정리 중 중요한 것 삭제 위험(복구 가능하다고 안내).

## 4. 부서별 특성 (교육 난이도 & 접근)

- **경영기획팀(서지애·박성희):** 수치 중심, 더존에 수년치 일관 축적, "정답이 있는" 데이터. → **자동화 상대적으로 쉬움.** 이미 앞서 있음.
- **홈쇼핑사업팀(박정선·신은선·서희정·양승혜):** 플라이마트(fmart)로 단순 주문취합·발주·정산만. 데이터 축적 구조 약함 → 단기 자동화 어려움. → **OMS 시스템 개발/유지보수로 해결**(심화 트랙 = 이 팀). **아웃풋 최상 예상**(공유 가능한 시스템이 답이므로).
- 기초 트랙 각 1명(경영기획·전략사업·콘텐츠사업·편성)은 부서 전파자 역할.

## 5. 공통 페인포인트 & 인사이트

- **"내 컴퓨터에서만 AI가 동작"** → 팀 공유 난망(구글드라이브 경로·세팅 동기화 필요). → **웹 시스템(OMS)이 근본 해결**: 내가 만들어도 퇴사 후 회사에 남는다. = AI의 최종 목적지.
- **국내/중국 SaaS의 MCP 미제공**(네이트온·카톡·위챗) → 자동 연동 한계. 구글·MS 계열은 가능 → 슬랙 이전 권장 배경.
- **금융권 API / 더존 내부 DB** → 개인 접근 어려움. RPA 클릭 자동화가 현실적 상한.
- **토큰/비용 관리** 니즈(서지애·서희정 프로플랜 한도) → 모델 선택·컨텍스트 최적화 교육 포인트.
- **봇 사전 교육(L1~L3) 현황:** 서버·코드관리 개념부터 체감 난이도 급등. 인터뷰 前 완료 목표였으나 7명 중 1명만 L1 완료. → 기초(IT/CS) 보강 필요.

## 8. v2 추가 — 기초 트랙 3명 인터뷰 (2026.07.09)

### 박소연 사원 (전략사업팀 · 기초) — 결과물 1~2단 / 이해 1단
- **업무:** 광고 영업 지원(영업사원=타 법인 소속·카톡 소통), 닐슨 효율, 운행표(프로그램 방송 광고 편성), 보고서, 큐시트·거래명세표.
- **도구:** GPT·Gemini (기본/무료 수준). 클로드 미사용.
- **활용:**
  1. 보고서 분석 — 원래 자료 하나하나 직접 찾았는데, 보고서 올리고 "수치 집중 분석" 지시. 프롬프팅 직접.
  2. 큐시트→거래명세표 변환 — 엑셀 2개(큐시트·거래명세) 업로드 후 양식대로 옮기기. **한 달 넘게 반복 교정해 본인 수작업과 동일 결과 도달.** 명령 규격화는 "어떻게 물어보면 돼?"라고 AI에 물어 받은 걸 복붙(자체 고안).
- **메신저:** 영업사원·팀장 모두 카톡. 사내 메신저(아마란스)는 공지용만.
- **인식/걱정:** 개인 과제 부담. 업무가 전화·카톡으로 요청 수신이라 자동화 어렵다고 봄(타당). "AI 뭘 쓰면 좋을지 계속 고민하는데 설명하기도 어렵다."
- **기대:** 보고서 작성·분석 보완, 거래명세표 효율화.
- **진행자 판정:** 1~2단 사이(대화~연결).

### 이현주 사원 (편성팀 · 기초) — 결과물 1~2단 / 이해 1단
- **업무:** 편성 스케줄(시간대별 프로그램), 방송 운행, 매월 특집 기획, 시청률 분석.
- **도구:** GPT(주, 회사 지원 프로) + Gemini(GPT 대답 안 좋을 때, 무료). 클로드 미결제 — **자기개발비 소진으로 결제 여력 없음(병목)** → GPT 투트랙 교육 검토.
- **활용:** 아이디어 기획·프로그램 소개·시청률 분석 포인트. 연결 안 함 — **매 대화 시작 시 상황 설명을 직접 입력**(수동 맥락 주입, 필요성은 스스로 체득).
- **반복업무:** 매일 시청률 정리 — 3개 채널 기입 폼에 주간/일 단위. **자동화·규격화 후보.** 폼 샘플 공유는 보안 확인 대기.
- **질문:** "기초 과정 끝내면 어디까지?" → 목표 합의: "내 업무+AI만 놓고 봤을 때 더 효율적으로, AI 알고 쓴다까지."
- **진행자 판정:** 1~2단 사이.

### 김현지 사원 (콘텐츠사업팀 · 기초) — 결과물 1단 / 이해 1~2단
- **업무:** 해외 콘텐츠 판권 구매 → 국내 OTT·IPTV 유통(유통 담당). 매달 정산 + 플랫폼 제공 소스(줄거리·로그라인 등 마케팅) + 외부 편집업체 디렉션.
- **활용:**
  1. 정산 — **보안 우려로 로우데이터는 AI에 제공 안 함.** 정산 자체는 엑셀 함수로 직접, AI는 **정제된 품의 양식만 주고 검증용**으로만. (로우/정제 데이터를 구분해 제공하는 판단 — 전원 중 유일)
  2. 콘텐츠 소스 작성 — 자동화 시도 후 실패(작품별 특성·뉘앙스). 프롬프트를 세세히 주고 복수 안 받아 직접 다듬는 방식 정착.
  3. 편집 디렉션 — 1차 결과물 캡처를 AI에 주고 변형 시안 몇 개 받아 업체 전달.
  4. 일상 검색(부동산 계약 등).
- **핵심 니즈:** **AI 학습·데이터 보안의 정확한 가이드라인** — 어디까지 넣어도 되는지. 진행자가 학습 정책 사실 검증 후 교육 항목으로 채택, **팀 공유 희망.**
- **인식:** "내 업무 특성상 자동화 어렵다" 강함 — 부분 타당. 진행자: 자동화가 전부 아님, 업무별 최선 단계가 있다.
- **진행자 판정:** 1단(대화).
