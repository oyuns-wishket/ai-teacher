# ai-teacher — AI 에이전트 학습 & 교육 커리큘럼

개인 학습 정리와 사내 교육 커리큘럼을 함께 관리하는 레포지토리입니다.

## 프로젝트 구조

```
ai-teacher/
├── notes/                  ← 학습 노트 (자유 형식 md, 빠르게 정리)
│   ├── claude-code/        ← Claude Code 관련
│   ├── codex/              ← OpenAI Codex 관련
│   ├── agent-patterns/     ← 에이전트 아키텍처/패턴
│   └── use-cases/          ← 도메인/업무별 적용 사례
├── curriculum/             ← 교육 커리큘럼 설계
│   └── outline.md          ← 전체 로드맵
├── materials/              ← 완성된 교육자료 (standalone HTML)
│   ├── 01-*.html
│   └── 02-*.html
├── CLAUDE.md
└── .claude/rules/
```

## 자동 워크플로우

사용자가 대화 중 학습 내용을 공유하면(직접 설명, 유튜브/블로그 내용 전달, 질문 등), 아래 흐름을 자동으로 수행한다.

### Step 1: 노트 자동 저장
- 대화에서 새로운 학습 내용이 나오면 → 해당 주제의 `notes/` 하위 폴더에 md 파일로 저장
- 기존 노트가 있으면 업데이트, 없으면 새로 생성
- 저장할 때 사용자에게 "notes/에 정리해둘게요" 정도로 간단히 알려주기
- 파일명 예: `notes/claude-code/context-window-원리.md`, `notes/use-cases/코드리뷰-적용.md`

### Step 2: 커리큘럼 제안 (허락 필요)
- 특정 주제의 노트가 3개 이상 쌓이거나, 하나의 큰 주제가 체계적으로 정리되면
- "이 내용 커리큘럼에 추가할까요?" 라고 제안
- 사용자가 승인하면 `curriculum/outline.md`에 항목 추가

### Step 3: 교육자료 제안 (허락 필요)
- 커리큘럼의 특정 주제에 대해 노트가 충분히 쌓이고, 교육자료로 만들 만큼 내용이 무르익으면
- "이 주제 교육자료(HTML)로 만들까요?" 라고 제안
- 사용자가 승인하면 `materials/`에 HTML 제작 (디자인 규칙 `.claude/rules/` 따름)

### 판단 기준
- 노트 저장: 항상 자동 (별도 허락 불필요)
- 커리큘럼 추가: 제안 후 허락받고 진행
- HTML 제작: 제안 후 허락받고 진행
- 커밋/push: 작업 완료 후 사용자에게 확인받고 진행

## 빌드/실행

정적 HTML. 브라우저에서 직접 열면 됩니다.

```bash
open materials/01-agent-개발-기본원리.html
```

## 컨벤션

- notes: 자유 형식 마크다운. 완성도보다 속도 우선.
- materials: standalone HTML. 외부 의존성 없이 `<style>` 인라인. 번호 프리픽스로 순서 관리.
- 파일명: 한국어 허용, 공백 대신 하이픈
- 참고자료: 공식 문서, 학술 논문(arXiv), 신뢰할 수 있는 블로그만 사용. 출처 URL 필수.
