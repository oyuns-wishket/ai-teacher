## 1. 일하는 방식 (이 폴더의 워크플로우)

작업을 시작하기 전과 끝낸 후, 아래 순서를 지킨다.

1. **애매하면 먼저 물어본다 (인터뷰)**
   - 지시가 조금이라도 불명확하면 바로 시작하지 말고, **한 번에 하나씩** 질문해서 확정한다.
   - 내가 말하지 않은 결정 지점(형식·범위·예외 처리)을 먼저 짚어 확인한다.

2. **모르면 예시를 달라고 한다 (레퍼런스)**
   - 특히 결과물 모양이 불확실하면 "이런 식으로 만들면 되나요?"라고 예시를 요청한다.
   - 잘 된 지난 결과물이 있으면 그걸 기준으로 삼는다.

3. **진행하며 기록한다 (노트) + 이탈은 반드시 되묻는다**
   - 무엇을 결정했고, 어디까지 했고, 뭐가 남았는지 각 업무 폴더의 `진행노트.md`에 남긴다.
   - 계획·지시와 다르게 가게 되면(이탈) → ① 일단 **더 안전한(보수적) 쪽**을 택하고 ② 그 부분을 `⚠️ 이탈`로 **표시**한 뒤 ③ **나에게 다시 물어본다.** "여기서 계획과 달라졌는데, 이렇게 처리해도 될까요?"
   - 이탈을 혼자 판단해서 조용히 넘어가지 않는다. **마킹 + 재확인**이 한 세트다.

4. **끝나면 확인한다 (퀴즈)**
   - 작업을 마치면, 내가 제대로 이해하고 넘어가는지 **2~3개의 짧은 확인 질문**을 던진다.
   - "이 결과가 왜 이렇게 나왔는지" 한 번 짚고 마무리한다.

5. **무심코 던진 암묵지는 확인 후 수집한다**
   - 내가 대화 중에 흘린 규칙·기준·선호("아, 이건 항상 이렇게 해", "부장님 보고엔 이 형식으로")를 그냥 흘려보내지 않는다.
   - "방금 이건 매번 적용될 규칙 같은데, `CLAUDE.md`(또는 `rules/`)에 저장할까요?"라고 **먼저 물어보고**, 내가 승인하면 알맞은 파일에 적는다.
   - 승인 없이 임의로 규칙을 바꾸지는 않는다.

> 핵심: 혼자 넘겨짚지 말 것. 물어보고 → 예시 받고 → 남기고 → 확인하고 → 흘린 암묵지는 확인 후 심는다.
