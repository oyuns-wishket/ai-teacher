# CLAUDE.md — 정산(매출 현황) 업무 규칙

> 이 폴더는 **채널별 매출 정산·현황** 업무용입니다.
> 이 파일의 규칙은 루트 `CLAUDE.md`보다 **우선** 적용됩니다.

## 이 업무는 무엇인가

- 한 줄 설명: 각 판매 채널의 주문·매출을 취합해 **채널별 매출 현황**을 만든다.
- 주기: 매일(일계표) · 매월(월 정산 현황).
- 최종 목적: 채널별 매출·마진이 한눈에 보이는 현황판.

## 데이터 구조 (일계표 기준)

`재료/일계표_샘플.xlsx`의 컬럼:

| 컬럼 | 뜻 |
|---|---|
| 발주업체명 | 판매 채널 (네이버·지마켓·쿠팡 등) |
| 상품ID / 상품명 | 판매 상품 |
| 카테고리 | 식품 · 주방 · 생활 · 뷰티 · 패션잡화 |
| 매출량 / 매출액 | 판매 수량 / 금액 |
| 반품량 / 반품액 | 반품 수량 / 금액 |
| 실매출액 | 매출액 − 반품액 |
| 매입단가 / 매입총액 | 원가 |
| 마진총액 / 마진율(%) | 실매출 − 매입총액 |

## 결과물 규칙

- **채널별 · 카테고리별**로 나눠서 집계한다.
- 매출은 항상 **반품을 제외한 "실매출"** 기준으로 본다.
- 완성된 현황판은 `재료/`가 아니라 이 폴더 바로 아래에 `결과-YYYYMMDD` 이름으로 저장한다.
- 기본으로 뽑을 것: ① 총매출·총마진 요약 ② 채널별 매출 ③ 카테고리별 비중 ④ 상품 TOP 10 ⑤ 채널별 마진율.

## 이 업무에서 조심할 것

- 원본 엑셀의 숫자를 임의로 고치지 않는다. 이상하면 나에게 먼저 알린다.
- 세일코리아넷은 **한 번에 10일치씩만** 다운로드된다 → 한 달은 여러 번 나눠 받은 파일을 합쳐야 한다.
- 세일코리아넷 "주문매출"과 자사 "편성매출"은 상품명이 달라, **매핑 테이블**(좌: 세일코리아 상품명 / 우: 편성명)로 맞춰야 한다.
- 상세 규칙은 `rules/` 참고.
